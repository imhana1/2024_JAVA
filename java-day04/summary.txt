Method(메소드)
    한가지 목적(기능)을 처리하기 위한 명령어 묶음(함수).
    메소드는 객체 안의 있는 함수의 명칭.

    4가지 유형 - 자바스크립트와 동일.
    1. 파라미터, 반환값 모두 사용.
    2. 반환값만 사용.
    3. 파라미터만 사용.
    4. 파라미터, 반환값 모두 사용하지 않는 유형.

    메소드를 사용하는 장점
    1. 반복된 코드의 사용을 줄일 수 있다.
    2. 코드의 관리가 수월해 짐.(유지/보수)

    문법>
    [접근제어자 기타제어자] 자료형 메소드명(파라미터목록)  -> 선언부(head)
    {
        실행문장들;                                  -> 정의부(body)
        [return] [결과값];
    }

    자료형 - 반환형(Return type) : 반환결과값의 자료형.

    반환값이 없을 때 자료형에 사용하는 키워드 - void(없다)

    return : 흐름을 되돌리다. 메소드가 종료.

static : 정적.(반대말 dynamic, 동적)
    자바에서 static은 인스턴스 없이 쓸 수 있다는 의미.

    자바는 객체지향 언어이기 때문에 모든 코드는 인스턴스(객체)화
    되어야 사용할 수 있다.
    즉, 프로그램 실행은 main을 포함하는 클래스(Main class)도
    인스턴스로 생성되어야 한다는 말임.
    그런데, Main class는 누가 인스턴스로 만드는가?(치킨 게임)
    static은 아무것도 없는 세상에 닭을 만드는 방법을 제공.

Method 구분
1. static method : static 키워드가 붙은 메소드.(main)
        인스턴스(생성) 없이도 호출(사용) 가능.(클래스 메소드)
2. instance method : 인스턴스가 생성되어야(new 사용) 존재하는 메소드.
        static 키워드가 안 붙은 메소드.

변수 용어 정리
1. 지역 변수 : 메소드(함수) 내부에 작성된 변수.
            메소드가 호출되면 생성되었다가, 메소드 종료 시 제거된다.
            (다시 메소드가 호출되면 새로 생성)
            초기화가 필요한 변수.
2. 매개 변수 : 메소드의 파라미터를 받기위해 작성된 변수.(지역 변수)
3. 참조 변수 : 인스턴스의 주소를 저장하는 변수.(예, String, Scanner)
4. 멤버 변수 : 클래스 내부, 메소드 외부에 작성된 변수.
            (자바스크립트이 전역 변수의 개념을 일부 차용)
            클래스 안의 모든 메소드에서 함께 사용할 수 있는 변수.
5. 인스턴스 (멤버) 변수 : 멤버 변수이며, static이 붙지 않는 변수.
            (인스턴스가 생성되어야 존재하는 변수)
6. 필드(field) : 인스턴스 멤버 변수를 지칭하는 다른 말.
7. 스태틱 (멤버) 변수 : 멤버 변수이며, static이 붙는 변수.
            (인스턴스가 생성되지 않아도 존재하여 사용할 수 있는 변수)
8. 클래스 변수 : 스태틱 변수를 지칭하는 다른 말.
9. 전역 변수 : 프로그램의 모든 영역에서 사용할 수 있는 변수.
            (자바에서는 쓰이지 않는다!)

메모리 영역(스태틱 영역, 콜 스택 영역, 힙 영역)별 변수 구분
1. 스태택 영역 : 클래스 변수. 참조 변수(static 키워드가 붙은 변수)
2. 콜 스택 영역 : 지역(매개) 변수. 참조 변수.
3. 힙 영역 : 인스턴스 변수(필드), 참조 변수.

객체 = 멤버변수 + 메소드 = class

스태틱과 인스턴스 사용 주의사항
1. 인스턴스 변수(메소드)는 스태틱 메소드 내에서 사용할 수 없다.
2. 인스턴스 메소드는 인스턴스 변수(메소드)와 스태틱 변수(메소드)
   모두 사용할 수 있다.
3. 스태틱 메소드에서 스태틱 멤버(변수/메소드)만 사용할 수 있다.

스태틱 변수는 모든 인스턴스가 공유하여 사용할 수 있다.
(스태틱 변수에 저장된 값은 모든 인스턴스에 동일하게 적용된다.)

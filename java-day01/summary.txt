자바 개요
모든 프로젝트(프로그램)은 하나의 main() 메소드가 있어야 하며, main() 메소드는 한 프로젝트 당 하나씩만 만든다

파일 생성 -> class 를 생성한다.(xxx.java)
java 프로그램은 class 단위로 작성.

HTML 에서의 class : 요소의 분류하거나 묶는 키워드
java 에서 class : 하나의 코드 묶음 단위

java 는 객체 지향 프로그래밍 언어
객체 지향 프로그래밍은 객체 단위로 작업
용어 정리
    1. 객체 (Object) : 개념 ("자동차", "에어컨")
    2. 인스턴스 (Instance) : 객체를 실체화 한 것 ("자동차-쏘나타")
    3. 클래스 (Class) : 객체를 정의 한 것 ("쏘나타의 설계도")
        -> 개발자가 작성하는 코드는 설계도

        인스턴스가 만들어지는 장소는 메모리

     한 클래스(설계도)로 여러 인스턴스를 생성할 수 있다
     프로그래밍 언어의 문법이란 설계도를 작성하는 방법
     객체 = 정보 + 기능
          = 변수 + 함수(메소드) = 클래스(class)

    객체 지향 프로그래밍 언어 학습 순서
    변수 (자료형) -> 연산자 -> 제어문 -> 객체 지향 관련

    변수 - 데이터를 담기 위해 일정 크기의 메모리 공간을 이름을 붙여서 사용하는 것 (값을 변경할 수 있다)
        !일정 크기!
        java 의 변수는 반드시 크기 (data type, 자료형)를 지정해야 함
        한 번 타입이 지정되면 변경할 수 없다
        (숫자를 저장하는 변수는 숫자만 저장할 수 있다)

        자료형 - 기본 자료형, 참조 자료형
        기본 자료형 (Primitive type)
        1. byte - 1byte 공간을 생성하는 타입(정수)
        2. short - 2byte (정수)
        3. char - 2byte (문자, 정수) - 알파벳 1글자
        4. int - 4byte (정수)
        5. long - 8byte (정수)
        6. float - 4byte (실수)
        7. double - 8byte (실수)
        8. boolean - 진리형 (true/false) 크기를 논하지 않음

        참조 자료형 (Reference type)
        String - 문자열 (크기 없음)
        인스턴스의 위치를 저장하는 변수를 만드는 자료형

        문자열 입력 방식
        JavaScript - '', "" 모두 사용 가능
        SQL - ''를 사용 (""를 사용할 수 있지만 ''를 권장)
        Java - "" 만 사용 (''는 char 타입의 데이터만 처리)
